# OctoAcme Role Accountability Checklist

Use this checklist during planning, sprint kickoff, and release readiness reviews to ensure ownership is explicit and handoffs are clear.

---

## Project Setup

- [ ] Project Manager and Product Manager are explicitly named
- [ ] Technical Lead / Architect is assigned
- [ ] QA/Testing Specialist is assigned
- [ ] Release/DevOps Engineer is assigned
- [ ] Security/Compliance Officer involvement is defined
- [ ] Stakeholder/Sponsor decision owner is identified

---

## Planning Readiness

- [ ] Business outcomes and success metrics are documented (Sponsor + PdM)
- [ ] Scope and priorities are approved (PdM + PM)
- [ ] Technical risks and dependencies are logged with owners (Tech Lead + PM)
- [ ] Test strategy is defined for critical flows (QA + Developers)
- [ ] Security/compliance requirements are captured (Security + Tech Lead)
- [ ] Release constraints and environment needs are documented (DevOps + PM)

---

## Execution Cadence

- [ ] Daily/weekly communication cadence is established
- [ ] Blocker escalation path is documented with time-to-escalate expectations
- [ ] Defect triage owner is assigned and active
- [ ] Design and code review responsibility is clear
- [ ] Risk register is reviewed weekly with updates and mitigation actions

---

## Release Readiness

- [ ] Acceptance criteria are verified complete (PdM + QA)
- [ ] CI checks are passing (Developers + DevOps)
- [ ] Security findings are resolved or formally accepted (Security + Sponsor)
- [ ] Rollback plan and runbook are confirmed (DevOps)
- [ ] Stakeholder go/no-go decision owner is available (Sponsor + PM)

---

## Retrospective Follow-Through

- [ ] Retrospective actions are mapped to named owners
- [ ] Each action has a due date and success signal
- [ ] Improvements are reviewed in the next planning cycle
